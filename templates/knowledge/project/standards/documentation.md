# Documentation Standard

Documentation updates should be concise, current, and tied to real project behavior. Prefer links to canonical runbooks, commands, and generated evidence over duplicating large docs in prompts.

When implementation changes user flow, harness behavior, install behavior, validation commands, or package publishing behavior, update the relevant docs and knowledge entries in the same change.

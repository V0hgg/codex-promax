# Global Validation Defaults

A task is done when the requested behavior is implemented, relevant verification has run, and remaining uncertainty is stated plainly.

Prefer focused checks near the changed behavior first. Broaden validation for shared code, public interfaces, publish workflows, security-sensitive changes, or user-facing flows.
